Timeline - Service Request Workflow Process

  • New Request

    New Service Request is raised by User

  • Adding Service Request into System

    Raised service request by user id added to the system by Tech Support

  • Technician Assignment

    A Technician is assigned to raised request to look into it and resolve issue

  • Sent for Approval

    Proposed solution for issue by Technician is sent for approval to high-tier techs

  • Service Request work flow completed

    Once the solution is valid, then it is approved by high-tier techs and the technician completes the service request.

  • Request Closed

    Once the request completed, it is closed and updated in the system by technician.

  • User notified and survey is sent

    After the service request is closed, the user is notified about the status and end user feedback regarding service delivery is gathered
